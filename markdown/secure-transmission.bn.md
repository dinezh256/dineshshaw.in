## আপনার ডেটা বাইরে যাওয়ার আগেই ট্যাব বন্ধ হতে দেবেন না

এখানে একটি মজার দৃশ্যকল্প রয়েছে: ব্যবহারকারী একটি লম্বা ফর্ম পূরণ করে, তার মনোযোগ অন্যদিকে সরে যায়, এবং সে ট্যাবটি বন্ধ করে দেয়। আপনার ব্যাকএন্ড কখনই কিছু জানতে পারে না। আপনার অ্যাপের উপর নির্ভর করে, এটি সাধারণ বিরক্তি থেকে শুরু করে একটি গুরুতর ডেটা ইন্টেগ্রিটির (data integrity) সমস্যা হতে পারে।

আমি এই সমস্যার সম্মুখীন হয়েছিলাম যখন আমি এমন একটি ফিচারে কাজ করছিলাম যা কোনো পৃষ্ঠায় কাটানো সময় ট্র্যাক করে, এমন একটি জিনিস যা *ব্যবহারকারী পৃষ্ঠাটি ছেড়ে যাওয়ার ঠিক সেই মুহূর্তেই* ফায়ার করা দরকার, কোনো টাইমার দিয়ে নয়। সাধারণ উপায়গুলো এখানে কাজ করেনি।

### সাধারণ পদ্ধতি এবং এটি কেন ব্যর্থ হয়

আমার প্রথম চিন্তা ছিল শুধুমাত্র একটি `beforeunload` লিসেনারের ভিতরে `fetch` ব্যবহার করা:

```js
window.addEventListener('beforeunload', () => {
  fetch('/api/track', { method: 'POST', body: JSON.stringify(data) });
});
```

এটি নির্ভরযোগ্যভাবে কাজ করে না। ব্রাউজার পেজ আনলোড হওয়ার সময় async অপারেশনের জন্য অপেক্ষা করে না, তাই অনুরোধটি (request) শেষ হওয়ার আগেই বাতিল হয়ে যায়। আপনি হয়তো মাঝে মাঝে ভাগ্যবান হতে পারেন, কিন্তু আপনি এর ওপর নির্ভর করতে পারবেন না।

### Beacon API-এর আগমন

`navigator.sendBeacon()` API ঠিক এই কারণেই রয়েছে। এটি ব্যাকগ্রাউন্ডে একটি ছোট HTTP POST পাঠায়, এবং সবচেয়ে গুরুত্বপূর্ণ হলো, ব্রাউজার *গ্যারান্টি* দেয় যে পেজটি বন্ধ হওয়ার সময়ও এটি পাঠানো হবে। কোনো অপেক্ষা নেই, কোনো ব্লকিং (blocking) নেই।

```js
navigator.sendBeacon('/api/track', JSON.stringify(data));
```

এটুকুই। বাকি সব ব্রাউজার সামলে নেয়। এটি নন-ব্লকিং, পেজ বন্ধ হতে দেরি করায় না এবং সমস্ত আধুনিক ব্রাউজারে সাপোর্টেড।

### React-এ এটি যুক্ত করা

আমি কীভাবে `useEffect` ব্যবহার করে একটি component-এর ভেতরে এটি সেট আপ করেছি তা এখানে দেওয়া হলো:

```jsx
import { useEffect } from 'react';

const useBeaconOnLeave = (data) => {
  useEffect(() => {
    const handleBeforeUnload = () => {
      navigator.sendBeacon('/api/track', JSON.stringify(data));
    };

    window.addEventListener('beforeunload', handleBeforeUnload);

    return () => {
      window.removeEventListener('beforeunload', handleBeforeUnload);
    };
  }, [data]);
};
```

এটিকে একটি কাস্টম হুকে (custom hook) র‍্যাপ করুন, আপনি যে ডেটা ট্র্যাক করতে চান তা পাস করুন, এবং আপনার কাজ শেষ। return-এর ভেতরে ক্লিনআপ গুরুত্বপূর্ণ। যদি component-টি অন্য কোনো কারণে আনমাউন্ট হয় (উদাহরণস্বরূপ, SPA-এর মধ্যে নেভিগেশন), তবে আপনি চাইবেন না যে একটি পুরোনো লিসেনার ঝুলে থাকুক।

### কয়েকটি বিষয় মনে রাখতে হবে

**পেলোডের (payload) একটি সাইজ লিমিট আছে।** `sendBeacon` বড় ডেটার জন্য নয়। এটিকে ছোট JSON অবজেক্টে সীমাবদ্ধ রাখুন: সেশন ডেটা, অ্যানালিটিক্স ইভেন্ট, এই ধরণের জিনিস। এর মাধ্যমে আপনার অ্যাপের সম্পূর্ণ অবস্থা (state) পাঠানোর চেষ্টা করবেন না।

**এটি কেবল সত্যিকারের পেজ আনলোড হলেই ফায়ার হয়।** একটি React অ্যাপের ভেতরের রাউট (Route) পরিবর্তনগুলি `beforeunload`-কে ট্রিগার করে না, কারণ সেগুলি ব্রাউজারের দ্বারা নয়, React Router দ্বারা নিয়ন্ত্রিত হয়। আপনার যদি ইন-অ্যাপ নেভিগেশন ট্র্যাক করার প্রয়োজন হয়, তবে আপনাকে এর পরিবর্তে আপনার রাউটারের `history`-তে হুক করতে হবে।

**সার্ভারটিকে এখনও সুরক্ষিত থাকতে হবে।** ক্লায়েন্ট ডেটা পাঠাচ্ছে মানেই যে আপনি অন্ধভাবে এর ওপর ভরসা করবেন তা নয়। সার্ভার সাইডে অন্যান্য অনুরোধের মতোই এটিকে ভ্যালিডেট (validate), প্রমাণীকরণ (authenticate) এবং স্যানিটাইজ করুন।

### আপনার কখন সত্যিই এটি ব্যবহার করা উচিত?

সত্যি বলতে, খুব একটা নয়। বেশিরভাগ অ্যাপেরই এর প্রয়োজন নেই। তবে আপনি যদি এর মধ্যে কোনোটি করে থাকেন তবে এটি সম্পর্কে জানা ভালো:

- অ্যানালিটিক্স বা এনগেজমেন্ট ট্র্যাকিং (পৃষ্ঠায় সময়, স্ক্রল গভীরতা)
- অটো-সেভিং ড্রাফ্ট স্টেট
- সেশন শেষ হওয়ার ইভেন্ট লগ করা
- "তারা ছেড়ে যাওয়ার আগের শেষ কাজ" (last action before they leave) ধরণের যেকোনো ফিচার

এটি একটি খুব নির্দিষ্ট ব্যবহারের ক্ষেত্র, তবে আপনার যখন এটি প্রয়োজন হয়, তখন আপনার সত্যিই এটি প্রয়োজন। এবং এখন আপনি এই কাজের জন্য সঠিক টুলটি জানেন।
